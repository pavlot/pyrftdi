Simple TX/RX¶

/examples/rx_tx/ demonstrate the simplest way to transfer data between two RFM devices using FTDI.

In this example, there are two scripts: /examples/rx_tx/rx.py to receive data and /examples/rx_tx/tx.py to transfer data

Each script runs an infinite loop to transmit/receive data.

Payload is 5 byte length and contains magic sequence cafeb0baxx, where xx is a counter

Configuration parameters¶

After RFM device wired to FTDI, you need to provide a URL for the FTDI device. Check pyFTDI documentation how to obtain correct url.

For both rx and tx scripts this value controlled by varible

1FTDI_URL = 'ftdi://ftdi:232h:555551/1'  # Url for ftdi chip, used to control RX

Make sure to use correct URLs for TX and RX script. This example assumes that rx and tx devices connected to different FTDI boards.

Next parameter required for pairing:

1CE_PIN = 7                              # FTDI pin used to as CE for RFM device

This is FTDI GPIO pin used to send CE signal to RFM device

Rest of parameters are related to RFM chip config and explained in comments

1PIPE_NO = 0                             # Pipe number used to receive data
2ADDR_WIDTH = Rfm75AddressWidth.ADDR_5   # How many bytes used for address
3PIPE_ADDR = b'\x11\x22\x33\x22\x11'     # Addres for selected pipe
4RF_CHANNEL = 0x02                       # Operating Rf Channel
5DATA_RATE = '250ksps'                   # Transfer speed
6PAYLOAD_SIZE = 0x05                     # Static payload size up to 32b

Configure chip to transmit/receive data¶

The simplest mode to transmit data it is when a transmitter sends data into the air and does not care whether it was received. Typical configuration is:

TX Part

 1controller = FtdiRfm75Controller(ftdi_port, ftdi_gpio, CE_PIN, reg_controller)
 2
 3if(not controller.is_connected()):                                      # Make sure we have chip connected
 4    logging.error("RFMx controller not found, exit")
 5    exit(1)
 6
 7logging.info("Chip id: {}".format(controller.get_chip_id().hex()))      # Print connected chip ID
 8controller.config_ctrl.reset_config()                                   # Reset config to be sure we start from 0 walues
 9controller.config_ctrl.set_rf_channel(RF_CHANNEL)                       # Configure RF channel, be sure it is the same for transmitter and receiver
10controller.config_ctrl.chip_init(DATA_RATE)                             # This is RFM magic init, most iportant to set data rate the same om transmitter and on receiver
11# Activate features and disable any dynamic features
12controller.activate_features()
13controller.config_ctrl.disable_dynamic_payload()
14controller.config_ctrl.disable_dynamic_acknowledge()
15
16# Configure adress width up to 5 bytes
17controller.config_ctrl.set_address_width(ADDR_WIDTH)
18# Target receiver address
19controller.config_ctrl.set_tx_address(TX_ADDR)
20# Do not pollute air with our RF waves
21controller.config_ctrl.set_tx_power(Rfm75TxPower.TX_PWR_LOW)
22
23# In this mode tranceiver does not care whether packet was received by target device
24controller.config_ctrl.pipe_ctrl.disable_auto_acknowledge()
25
26# Enable hardware CRC calculation and set it to 2 bytes
27controller.config_ctrl.crc_ctrl.set_crc_len(Rfm75CRCLen.CRC_2)
28controller.config_ctrl.crc_ctrl.enable_crc()
29
30# Switch to TX mode
31controller.set_mode_tx()
32# Turn on RFM power. Note this is standby mode. Data is not transmitted until CE enabled and data present in TX buffer.
33controller.power_up()

After this call

1controller.write_tx_payload(payload)

to send a payload.

Receiver initialisation almost the same as for transmitter, except few details:

First pipe should be configured:

1controller.config_ctrl.pipe_ctrl.enable_pipe(PIPE_NO)   # Enable required pipe
2controller.config_ctrl.pipe_ctrl.set_rx_pipe_address(PIPE_NO, PIPE_ADDR)    # Set pipe address, RFM dewice has 5 pipes, each must have uniq address
3controller.config_ctrl.pipe_ctrl.set_rx_pipe_payload_width(PIPE_NO, PAYLOAD_SIZE)   # Static payload size
4controller.config_ctrl.pipe_ctrl.disable_auto_acknowledge() # Disable AA

After initialisation is done, the chip must be set to RX mode using following calls:

1controller.set_mode_rx()
2controller.power_up()
3controller.ce_on()

Remember about ce_on() after power_up() - this will instruct RFP device to listen air for incoming data.

When data available controller.read_rx_payload_len() return non-zero value, so typical receive loop will be like:

1payload_len = controller.read_rx_payload_len()
2if(payload_len > 0):
3    logging.info("Data received: {}".format(controller.read_rx_payload(payload_len).hex()))
